> 一款讲述**中国传统事物**走向世界的选集式叙事收藏游戏。
> 第一章：福建老酒。纯 2D RPG 探索 · 叙事选择 · 道具与徽章全部上链。
>
> AdventureX 2026 参赛项目 ｜ 文档版本 v2.0 ｜ 更新：2026-07-23

## 0. 一句话介绍

> **玩一遍一件中国传统事物的故事，你就真正读懂了它——读懂的那一刻，它凝成一枚属于你的链上徽章。**

《拾遗》是一部选集式（anthology）叙事游戏：玩家扮演一个海外/年轻的"拾遗人"，每一章走进一件正在被世界淡忘的中国传统事物的记忆里，用"亲历它的故事"真正理解它，最后把它凝成一枚数字徽章收进收藏馆。沿途的信物与最终的徽章全部铸造在 Injective 链上——玩家真正**拥有**了他读懂的这份文化。

**第一章做福建老酒，但游戏讲的是"中国传统事物"这个大主题，后续章节是茶、瓷、丝、非遗、文旅……酒只是首个案例。**

## 2. 游戏身份与基调

- **基调立在"中国传统事物"，不是"酒"。** 主界面是一座收藏馆，不是酒馆。
- **选集式结构**：章节 = 一件传统事物。第一章福建老酒，后续茶/瓷/丝/非遗以锁定剪影呈现。
- **情感内核**：拾起正在被世界遗忘的东西，让它以被"读懂 + 拥有"的方式重新活过来、走向全球。
- 名称：**《拾遗 · Gleanings》**（"拾遗"= 拾起遗失/遗产，天然覆盖非遗与所有传统事物）。备选《回声》《遗光》。

## 3. 世界观与叙事框架

### 3.1 出海框架（把 SuperZygo 命题写进剧情）

一个在海外的年轻人，偶得/继承一件落灰的中国传统物件。当他触碰它，物件的记忆把他拉进一段旅程——他一段段亲历这件事物的身世，从"这是什么奇怪的东西"变成"我读懂了、我珍惜它"。旅程走完，这件事物在他手里凝成一枚数字徽章，永远收藏。

> 传统产品不靠贴符号、翻译故事出海，而是让世界**亲历**它的故事而被接住。这就是 WOPC 命题的答案。

### 3.2 章节 / 幕 / 信物层级

```
章节（= 一件传统事物）
  └─ 幕（= 这件事物身世的一个片段）── 通关得【ERC-721 信物】+ 一段故事
  └─ 幕
  └─ 幕
  └─ 结局 ── 集齐信物 → AI 文化酒签 → 铸造【ERC-721 徽章】
```

- **章节** = 福建老酒 / 茶 / 瓷 …（demo 只做福建老酒）
- **幕** = 3–4 幕的叙事关卡
- **信物** = 每幕收集的链上藏品
- **徽章** = 每章通关铸造的最终链上藏品

## 4. 核心玩法（纯 2D RPG 探索 + 叙事选择）

### 4.1 设计定位

玩家在大于视口的 tile map 中走位、与 NPC 交谈、检视器物、接取并完成短任务，
再通过叙事选择改变 lore、信物与结局语气。**不做**战斗、不做经营、不做开放世界；
游戏性集中在探索、任务、背包、互动对象、场景状态变化与收藏。

### 4.2 通用章节模板（对任何传统事物复用）

每一幕包含：
1. **进入地图**：主角被拉进记忆，摄像机跟随玩家，只显示完整地图的一部分。
2. **探索与对话**：沿可读路径寻找 NPC、器物和发光交互点，获得任务与文化线索。
3. **任务与背包**：收集或使用 1–3 个关键物件，触发投料、开坛、端汤等无失败惩罚的生活动作。
4. **选择驱动**：选择决定解锁哪段 lore、拿到哪枚信物与结局情感基调；分支保持短而可回收。
5. **场景变化**：任务完成后物件、NPC、灯光或出口状态改变，让玩家看见行动结果。
6. **幕末结算**：获得一枚 **ERC-721 信物** + 一段故事。
7. **章末**：选择与信物汇成受控 AI 双语文化酒签 → 写入 metadata / `pathHash` → 铸造该事物的 **ERC-721 徽章**。

### 4.3 游戏性来源（为什么"够玩"而非只是读文字）

- **有意义的选择**：每一幕都要玩家做出视角/情感抉择，带来不同 lore 与信物，产生"再玩一遍看别的分支"的动机。
- **空间探索**：地图至少为视口的 1.6–2.5 倍，玩家通过走位建立对酒坊、灶间和收藏馆的空间记忆。
- **短任务闭环**：NPC 指引 → 找到物件 → 使用/交付 → 场景状态变化，单个任务 2–5 分钟。
- **收集驱动**：信物图鉴 + 徽章墙，稀有信物需特定选择路径才能拿到。
- **生活动作的仪式感**：亲手投料、开坛、斟酒等动作，把"读文化"变成"做文化"。

## 5. 第一章 · 福建老酒（逐幕脚本）

> 底蕴取自福建老酒真实文化：红曲黄酒、冬酿节气、月子酒人情、窖藏时间。

### 幕一《开坛》（引子，简短）
- 场景：海外公寓，主角开一坛落灰的福建老酒。酒香升起，画面被拉入记忆。
- 选择：你先被什么吸引？【香气】/【那一抹红】/【坛身的温度】→ 决定后续叙事切入口与旁白语气。
- 产出：进入记忆，无信物。

### 幕二《冬酿》
- 场景：旧时福建冬日酒坊，霜降与冬至之间。
- RPG 任务：在 40×28 酒坊地图中找到阿凤师，检视红曲盘与陶坛，领取红曲样品，
  依次完成投料、封坛；任务后红曲盘清空、陶坛加封、出口开放。
- 选择：向酿酒人问一个问题——【红曲是什么】/【为何非冬天不可】/【这酒给谁喝】→ 解锁不同 lore 片段。
- 产出信物：**冬酿曲印**（NFT）。

### 幕三《月子酒》（情感高点）
- 场景：一户福建人家，产妇初为人母，灶上炖着老酒鸡汤/姜母鸭。
- 叙事：体会这坛酒在福建家庭里代表的祝福、照护与人情。
- 选择：如何回应这份人情 → 决定结局情感基调（温暖/怅惘/敬意）。
- 产出信物：**青花酒盏**（NFT）。

### 幕四《归》（结局）
- 场景：回到现代海外，主角面对同一坛酒，态度已变。
- 结算：集齐【冬酿曲印】+【青花酒盏】→ 铸造 **福建老酒徽章**（NFT），收进收藏馆。
- 多语言：中英双语呈现，供海外玩家体验。

**MVP 范围**：幕一做成简短引子；完整做**幕二冬酿 + 幕三月子酒 + 幕四结局徽章**。

## 6. 收藏馆 / 图鉴系统

- **主界面 = 可探索收藏馆**：在 40×24 地图中走到展柜前检视；福建老酒徽章点亮，
  旁边**茶、瓷、丝、非遗**为灰色锁定位与编号。

## 7. 区块链设计（Injective）

### 7.1 上链 / 链下分工

| 内容 | 位置 | 理由 |
|---|---|---|
| ERC-721 信物、徽章、`pathHash`、metadata URI | **链上（Injective 测试网）** | 需要真实拥有权与可验证的玩家旅程 |
| 剧情、选择、轻互动、进度、图鉴逻辑 | 链下（本地/前端） | 省 Gas、逻辑清晰、开发快 |

> 向评委的解释：只有"藏品"上链，因为玩家真正拥有了他走过、读懂的这份文化；其余无需上链。

### 7.2 技术选型

- **Injective EVM（MultiVM）+ Solidity**——不碰 Rust/CosmWasm，降低门槛。
- 参考 docs.injective.network 官方 SDK / EVM 文档。
- 测试网部署即可满足赛道要求。

### 7.3 合约设计（务实版）

**方案：单个 ERC-721 合约 `Gleanings`**。每枚信物与徽章都是唯一藏品，可分别记录玩家选择路径、稀有度与个性化图；合约基于 OpenZeppelin v5，并实现 ERC-2981 版税接口。

核心接口：
```solidity
// 信物：玩家凭后端签发的 EIP-712 voucher 铸造
function mintRelic(RelicVoucher calldata v, bytes calldata sig)
    external
    returns (uint256 tokenId);

// 徽章：链上校验调用者实际持有全部必需信物
function claimBadge(
    uint256 seriesId,
    uint256[] calldata relicTokenIds,
    string calldata badgeUri
) external returns (uint256 tokenId);

// 收藏馆读取
function collectionOf(address owner) external view returns (uint256[] memory);
```

- **防作弊**：后端在剧情完成、AI 结果落库后签发 EIP-712 voucher；合约校验玩家、章节、信物、签名和已使用 digest。
- **metadata**：`name`、`series`、`piece`、`story`、`image`、`rarity`、`verified_fact_ids`、`creative_expression`、`path_hash`。

### 7.4 钱包与交互

- 前端集成钱包连接（EVM 钱包 / Injective 支持的钱包）。
- 关键时刻（幕末、章末）触发铸造交易；其余全程无需上链，体验顺滑。

## 8. AI / AIGC 设计（全程无视频，控成本）

### 8.1 AI 用途与产品闭环

| 用途 | 技术 | 服务对象 |
|---|---|---|
| 构建期场景、人物、器物初稿 | 图像生成 → 人工清理 → 色板量化 | 统一的“闽地民俗标本馆”视觉 |
| 章末个性化文化酒签 | 已核对事实块 + 玩家选择 + LLM 结构化输出 | 海外可读的双语文化表达 |
| 藏品视觉组合 | 器物基底 + 铭文 + 稀有度 + 视觉母题 | 与玩家路径对应的唯一藏品 |
| 翻译与润色 | LLM 初稿 + 人工校对 | 中英叙事、文化卡与分享素材 |

产品闭环：

```text
玩家选择
  → 红曲朱印与 choiceId
  → 受控事实 + 情绪/文化焦点/视觉母题
  → AI 双语文化酒签
  → 玩家确认
  → metadata URI + pathHash
  → EIP-712 voucher
  → Injective ERC-721
```

AI 不自由编造文化事实。红曲、冬酿、月子酒等事实来自品牌方核对后的只读资料；AI 只组织表达、生成玩家铭文与视觉参数。界面明确区分“文化事实”“AI 创意表达”“链上记录”。

### 8.2 SuperZygo 成果

- **核心成果**：可玩的福建老酒 AI 文化酒签生成与收藏闭环。
- **辅助成果 1**：闽地民俗标本馆视觉系统与个性化信物组合器。
- **辅助成果 2**：中英双语文化卡、玩家旅程短文与海外分享卡。

### 8.3 Demo 稳定性策略

- 关键美术**预生成**并入包，保证 demo 流畅；
- 现场保留 **1 次实时文化酒签生成**作为 wow 时刻；
- 超时或服务失败时，按玩家情绪映射到温暖/传承/思念三套预生成结果，仍可继续铸造。

## 9. 技术架构

```
┌─────────────────────────────────────────────┐
│  前端（Web，浏览器即 demo 链接）                 │
│  · Phaser：tile map / 碰撞 / 摄像机 / NPC / 任务 │
│  · React：对白/选择、收藏馆、钱包与铸造 UI          │
│  · 钱包连接 + 铸造交互                           │
├─────────────────────────────────────────────┤
│  生成服务（构建期素材 + 1 次运行时生成）            │
│  · 已核对文化事实库 + 玩家选择                     │
│  · 结构化双语文化酒签 + 降级模板                   │
│  · metadata 固化 + EIP-712 voucher 签发          │
├─────────────────────────────────────────────┤
│  合约层（Injective EVM 测试网，Solidity）        │
│  · ERC-721 Gleanings：信物 + 徽章               │
│  · pathHash / metadata URI / 集齐校验           │
└─────────────────────────────────────────────┘
```

- **前端引擎**：使用 **React + Phaser 3**。Phaser 负责 tile map、碰撞、角色、
  摄像机和交互；React 负责钱包、铸造、设置和页面级 UI。
- `pixelArt: true`、`roundPixels: true`，CSS 使用 `image-rendering: pixelated`；
  游戏世界、摄像机和 UI 坐标全部取整数。
- **不用 RPG Maker**（接钱包困难）。
- 全程用 **Qoder** 脚手架与多智能体开发（满足 Qoder 赛道）。

### 11.2 SuperZygo 文书（推荐选 Q1 / Q3 / Q5）

> 叙事线：独特创意 → 硬核创新 → 可规模化商业价值。以下为初稿，按团队真实语气微调。

**Q1 · 创意出发点与价值**
> 灵感来自一个观察：中国传统事物出海的失败，几乎从不是翻译问题，而是"文化没有变成让世界想拥有的东西"。于是我们把"读懂一件中国传统事物"做成了一款可以真正玩的叙事游戏——玩家亲历它的故事，读懂的那一刻，它凝成一枚属于自己的链上徽章。福建老酒是我们的第一章。
> **一句话价值：把文化出海从没人看得懂的方案，变成一款边玩边读懂、并真正"拥有"中国传统文化的游戏，让任何传统事物都能像通关一样走向世界。**

**Q3 · 最大亮点与创新**
> ①**玩法即命题**：游戏核心是"亲历一件传统事物的故事"，玩的过程本身就是文化的重新表达与传播，而非玩完再做营销。②**选集化 = 可无限复制的出海方法论**：每件传统事物都是一个新章节，共用同一套叙事引擎与链上系统，demo 做福建老酒，架构天然可复制到茶、瓷、非遗。③**道具与徽章上链**：玩家沿途的信物、通关的徽章都是 Injective 上可拥有、可交易的数字藏品，并绑定真实文化背景故事，让中国文化第一次成为世界玩家真正"拥有"的东西。

**Q5 · 可复用性与商业化拓展**
> 我们从第一天就是"平台 + 章节"架构：叙事引擎、AI 生成管线、链上藏品系统是通用底座，每接入一件传统事物只需替换其文化母题、故事脚本与藏品资产，即成为新章节，福建老酒是首个验证案例。**复制成本极低**：换事物 = 换素材，引擎/合约/前端不动，数天即可上线新章节。**商业模式**：对品牌方按章节收取 AI 出海内容共创费，优秀产出直接进入打样与联合开发（福建老酒已承诺此路径）；对 C 端发行限量藏品并抽取二级交易分成。**拓展空间**：茶饮、食品、非遗、文旅、服装皆可复制，每个新品牌既是客户也是新内容，平台沉淀出一套"中国传统文化 AI 出海"的可复用方法论与文化藏品宇宙。

## 12. 风险与取舍

| 风险 | 应对 |
|---|---|
| 3 天时间紧 | 严守 MVP：仅第一章 2 幕 + 结局；预生成美术；叙事选择型省去重引擎 |
| Demo 现场 AI 不稳定 | 关键素材预生成入包，仅保留 1 次实时生成做 wow |
| 上链复杂度 | 单 ERC-721 合约、测试网、关键时刻才交易；voucher 与降级 metadata 预演 |
| AI 像制作工具而非产品能力 | 让玩家选择直接决定运行时双语文化酒签、藏品组合与 `pathHash` |
| SuperZygo 命题被叙事冲淡 | 事实库受控；生成结果面向海外读者；分享卡展示实际跨文化表达 |
| 文化事实被 AI 编造 | 事实层只读、创意层单独标记；上线前由品牌方审核 |
| 偏题禁赛 | 只报 3 个证据充分的主赛道；Qoder 未真实使用则撤；不报秒哒 |
