# 《拾遗》纯 2D 像素 RPG v2 设计

> 确认日期：2026-07-23
> 状态：用户已确认
> 取代范围：视觉表现、地图结构、角色表现与素材规格；叙事主题、文化事实边界与链上逻辑保持不变。

## 1. 改版原因

v1 使用高分辨率生成图模拟像素质感，存在三个问题：

1. 柔光、渐变与连续明暗让场景介于 2D 插画和 3D 渲染之间。
2. 高清图非整数比例缩小后，像素簇和轮廓不稳定，游戏尺寸下显得模糊。
3. 大背景与半身立绘只能支撑视觉小说，无法形成可移动、可探索的完整 RPG。

v2 改成严格的纯 2D 像素叙事 RPG。所有正式素材直接服从固定网格、色板和投影规则，不再把插画缩小后当作游戏素材。

---

## 2. 显示与像素基准

| 项目 | v2 规范 |
|---|---|
| 原生画布 | `640×360` |
| 桌面放大 | 3× 对应 `1920×1080` |
| CSS | `image-rendering: pixelated` |
| 地图格 | `32×32` |
| 人物单帧 | `32×48` |
| 主色板 | 全项目约 24 色 |
| 透明规则 | tile / sprite / item 使用硬边 alpha；FX 可半透明 |
| 禁止 | 抗锯齿、柔焦、景深、连续渐变、浮雕、写实材质和非整数缩放 |

地图采用固定 3/4 俯视投影。墙体、门窗、器物和人物必须遵循同一投影规则，不使用透视消失点，也不使用等距 2.5D。

---

## 3. 地图规模与镜头

地图大于屏幕，通过镜头跟随形成探索空间：

| 地图 | 像素尺寸 | tile 尺寸 | 核心区域 |
|---|---:|---:|---|
| 海外公寓 | `960×640` | `30×20` | 遗物箱、字条、酒坛、门口 |
| 冬日酒坊 | `1280×896` | `40×28` | 蒸米、红曲、陶坛、工作台、窗边 |
| 月子灶间 | `1024×768` | `32×24` | 灶火、食材、面线、床边 |
| 收藏馆 | `1280×768` | `40×24` | 信物墙、文化卡、锁定章节 |

镜头规则：

- 默认平滑跟随玩家，但最终坐标对齐整数像素。
- 到达地图边缘时停止滚动。
- 对话、投料、揭坛和铸造时可锁定镜头。
- 前景遮挡层在玩家经过时可切换透明或隐藏。

## 4. 完整叙事 RPG 循环

```text
自由探索
  → 检查器物 / NPC 对话
  → 获得任务与背包物品
  → 完成酿造或烹饪动作
  → 做出叙事选择
  → 地图状态改变
  → 获得信物与文化事实
  → 返回收藏馆
```

不加入战斗系统。RPG 完整性来自：

- 可滚动地图与碰撞；
- 四方向角色移动；
- NPC、交互物与任务；
- 背包、任务记录和世界状态；
- 地图对象随进度发生变化；
- 信物、文化卡与收藏馆成长。

玩家完成动作后必须看到世界变化，例如：

- 空坛变成封坛；
- 红曲托盘从“未取”变成“已取”；
- 灶台从冷态变成点火；
- 桌上出现完成的面线；
- NPC 位置、朝向和对白随任务推进改变。

## 5. 地图数据与状态

每张地图由五层组成：

1. `ground`：地面、雪地、室内基础铺面；
2. `structure`：墙、窗、门、固定家具；
3. `objects`：陶坛、桌面物品、灶台等；
4. `foreground`：梁柱、门框、蒸汽等遮挡层；
5. `events`：碰撞、触发区、出口、NPC 与交互物。

运行时状态：

```text
worldState      地图对象、门、火、坛等状态
questState      当前任务、步骤与完成条件
inventory       红曲、糯米、酒盏、文化卡等
choiceState     choiceId、emotion、culturalFocus、visualMotif
collectionState 信物、徽章与解锁章节
chainState      未同步、待签名、确认中、已上链、失败
```

断网、AI 或钱包失败不阻塞剧情通关。藏品先进入本地收藏馆，并显示“待同步”；恢复后再签发 voucher 与铸造。

## 6. 人物与动画

人物单帧为 `32×48`，行走表为 `96×192`：

- 横向三帧：左脚、站立、右脚；
- 纵向四行：下、左、右、上；
- 每人 12 帧基础行走；
- 关键角色增加 2–4 个剧情动作。

P0 人物：

- Yi：行走、揭坛、拾取；
- 少女太婆：行走、投料、封坛；
- 阿凤师：站立、说话、检查陶坛；
- 中年太婆：行走、端碗、点灶；
- Mia：行走、说话。

人物不再使用大幅半身立绘。表情通过 32×48 sprite 的头部像素、姿态和对话框头像完成。

## 7. v2 素材系统

```text
assets/rpg_v2/
├─ palette/
├─ previews/
├─ tilesets/
├─ maps/
├─ sprites/
├─ objects/
├─ items/
├─ ui/
├─ fx/
├─ audio/
├─ manifest/
└─ prompts/
```

| 类型 | 规格 |
|---|---|
| Tileset | `256×256` 或 `512×512`，32×32 网格 |
| Sprite sheet | `96×192`，硬边透明 |
| 地图器物 | `32×32`、`64×64` 或多格组合 |
| 背包图标 | `32×32` |
| 信物详情 | `128×128` |
| UI | 直接按 640×360 原生尺寸制作 |
| 实机截图 | `640×360`，可整数倍放大展示 |

v1 保留在原 `assets/` 路径，只作构图、材质和文化气氛参考。v2 不得直接引用 v1 大背景、半身立绘、浮雕信物或 UI 图。

## 9. 质量门槛

自动检查：

- 文件尺寸符合 manifest；
- tile 与 sprite 图集尺寸能被网格整除；
- 透明素材四角 alpha 为 0；
- sprite / tile / item 不含部分透明像素；
- 正式素材颜色均在项目主色板内；
- 色板数量不超过约 24；
- 预览只进行整数倍最近邻放大。

人工检查：

- 所有对象投影方向一致；
- 角色在地图中的比例稳定；
- 门、桌、陶坛等碰撞尺寸合理；
- 32×32 下仍能辨认器物；
- 无连续柔光、绘画笔触、3D 浮雕和伪像素纹理；
- 福建老酒场景与器物完成文化审核。
