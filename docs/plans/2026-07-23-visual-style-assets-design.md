# 《拾遗》视觉风格与素材系统设计

> 版本 v1.0 ｜确认日期：2026-07-23
> 已确认方向：**A｜闽地民俗标本馆**，保留少量复古叙事游戏反馈。

## 1. 设计目标

《拾遗》的视觉系统要同时完成四件事：

1. 让玩家感到自己翻开的是一册可触摸的民俗档案，而不是普通视觉小说或 Web3 页面。
2. 用统一的像素语言呈现人物与记忆，用纸、木、陶、酒液等材质建立文化真实感。
3. 让玩家选择留下可见痕迹，并真实进入 AI 生成、信物铭文与链上路径数据。
4. 建立可复用的“平台 + 章节”系统，未来替换茶、瓷、丝、非遗时不必重做界面骨架。

成功标准不是“看起来很中国”，而是玩家能从界面材料、光线、声音与动作中辨认出福建老酒的生活语境。

---

## 2. 核心视觉母题：闽地民俗标本馆

主界面是一册被玩家亲手翻开的民俗标本册。每一幕像一页活起来的档案：

- 像素场景负责记忆、人物与动作。
- 宣纸、漆木、陶釉、红布、酒液琥珀光负责触感。
- 对白、选择和文化解释像后人留下的批注、酒签与编号。
- 收藏馆像不对称的标本墙，展品带出处、年份、玩家铭文与链上证明。

视觉关键词：**温润、克制、可触摸、编辑感、生活史、时间沉淀**。

禁止直接采用的泛化符号：

- 大面积正红与亮金；
- 龙、凤、祥云、卷轴等无叙事依据的“国潮套件”；
- 仿古边框堆叠、满屏印章、整页书法字体；
- Web3 霓虹、币价感、金属科幻面板；
- 光滑 3D 卡牌、写实摄影与低清像素素材混用。

## 3. 颜色与材质

### 3.1 核心色板

| Token | 用途 | HEX |
|---|---|---|
| `ink-950` | 主背景、最深文字 | `#211A17` |
| `ink-700` | 次级文字、柔和阴影 | `#5B514A` |
| `paper-100` | 主纸面 | `#EADDC5` |
| `paper-50` | 高亮纸面 | `#F4EBDD` |
| `amber-500` | 酒液、记忆高光 | `#C9873F` |
| `hongqu-600` | 选择、印记、关键动作 | `#A83B32` |
| `lacquer-700` | 木框、漆面 | `#5B3A29` |
| `clay-500` | 陶坛与暖阴影 | `#7A5845` |
| `blue-500` | 文化出处、可信信息 | `#48677A` |
| `frost-300` | 冬日环境、锁定状态 | `#B7C2C0` |
| `straw-400` | 编号、稀有度辅助 | `#D4B46A` |

红曲朱只用于“玩家正在做的事”，青花蓝只用于“有来源、可查证的文化信息”。金色不承担主 CTA。

### 3.2 材质层

- 宣纸：对白、文化卡、生成结果与批注。
- 漆木：导航、收藏展框、章节页签。
- 陶釉：信物容器、按钮按压高光、铸造容器。
- 封坛红布：主视觉锚点、关键转场。
- 红曲颗粒：选择印记、像素溶解、生成过程。
- 酒液琥珀光：记忆层与现实层的过渡。
- 木格窗影：空间层次，不作为装饰纹样满铺。

## 4. 字体与排版

| 角色 | 中文建议 | 英文建议 | 用途 |
|---|---|---|---|
| 展示标题 | Noto Serif SC / 思源宋体 | Fraunces | 章节名、藏品名、主标题 |
| 正文 | MiSans / 系统人文黑体 | Geist | 对白、说明、按钮 |
| 像素标记 | Fusion Pixel 12px | 同系列等宽像素体 | 年代、编号、稀有度、链上状态 |

规则：

- 正文不使用像素字体；英文不整段使用全大写。
- 中文正文行高为字号的 1.65–1.8 倍，英文为 1.45–1.6 倍。
- 标题与正文至少形成 1.6 倍字号差。
- 每行中文控制在约 22–30 字，英文控制在约 45–68 字符。
- 禁用 Inter、Roboto、Arial、Open Sans、Helvetica 作为品牌主字体。

## 5. 页面架构

### 5.1 主叙事页

桌面端采用“横向档案页”：

- 左侧约 65%：场景、人物与轻互动。
- 右侧约 35%：章节编号、地点、年代、文化批注与进度。
- 底部：贴近画面的宣纸对白条。
- 选择出现时：三张略有错位的酒签覆盖在对白上方。

移动端在 768px 以下强制改成单列：

- 场景在上，对白与选择在下；
- 移除旋转、负边距和重叠；
- 触控目标不小于 44×44px；
- 不使用固定 `100vh`，按动态视口处理。

### 5.2 收藏馆

收藏馆采用不对称标本墙，而不是等宽卡片网格：

- 主信物占大展框；
- 次信物与文化卡错落排布；
- 每件展品有编号、出处、玩家铭文与链上状态；
- 未解锁内容仅显示纸页压痕、锁定编号和一句下一章引子。

### 5.3 生成与铸造

AI 生成与链上铸造是同一段连续仪式：

1. 回顾玩家留下的选择印记；
2. 显示“事实层”和“创意表达层”；
3. 生成双语文化酒签与视觉组合；
4. 玩家确认；
5. 钱包签名与铸造；
6. 信物进入收藏馆。

## 6. 核心组件

所有主要容器采用“双层展框”：

- 外层是漆木、纸托或陶釉承托，负责材质与阴影；
- 内层是内容纸面或像素画面，负责信息；
- 两层圆角同心，避免普通灰色细框和模板化投影。

P0 组件：

1. `StoryStage`：像素场景、人物、互动热点。
2. `DialogueSheet`：人物名、正文、继续提示、语言切换。
3. `ChoiceTag`：默认、悬停、键盘焦点、已选、不可用五态。
4. `CultureNote`：事实出处、双语短解释、查证状态。
5. `ChapterTab`：章节、幕、地点、年代。
6. `SpecimenFrame`：信物展示、锁定、稀有、选中四态。
7. `GenerationSheet`：选择摘要、生成中、结果、降级结果。
8. `MintPanel`：未连接、待签名、链上确认中、成功、失败五态。
9. `ProvenanceMark`：`文化事实`、`AI 创意表达`、`链上记录`三类标记。
10. `ShareCard`：中英双语、藏品图、玩家铭文、二维码/链接位。

## 7. 选择、AI 与链上数据流

选择不是临时按钮状态，而是一条贯穿体验的数据：

```text
酒签选择
  → 页面留下红曲朱印
  → 记录 choiceId / emotion / culturalFocus
  → 组合受控 AI 输入
  → 生成双语文化酒签与视觉参数
  → 生成结果写入 metadata
  → 选择序列生成 pathHash
  → voucher 签名
  → Injective ERC-721
```

AI 不允许自由编造文化事实。输入拆成两层：

- **事实层**：品牌方核对后的红曲、冬酿、月子酒等资料，只读。
- **创意层**：玩家选择、情绪、海外表达角度、铭文与视觉参数，可生成。

生成失败时使用相同选择映射到预生成模板，保证 Demo 能继续完成铸造。

## 8. 动效与声音

动效模拟纸、布、蒸汽与器物的重量，不使用金币、烟花和 Web3 霓虹。

| Token | 时长 | 用途 |
|---|---:|---|
| `quick` | 180–220ms | 按压、焦点、图标反馈 |
| `standard` | 320–450ms | 酒签抬起、纸页展开 |
| `narrative` | 1.2–1.8s | 入记忆、章节切换、藏品显影 |

统一缓动：`cubic-bezier(0.32, 0.72, 0, 1)`。

性能边界：

- 只动画 `transform` 与 `opacity`；
- 滚动区域不使用大面积实时模糊；
- 粒子同时显示不超过约 60 个；
- 支持 `prefers-reduced-motion`，关闭漂移、旋转和像素溶解；
- 关键声音均有字幕或视觉反馈，静音不影响通关。

## 9. 素材分层

1. **风格锚**：酒坊场景、少女太婆、冬酿曲印、酒签选择。
2. **场景层**：公寓、酒坊、灶间、收藏馆。
3. **人物层**：统一角色基准与少量表情覆盖。
4. **材质层**：纸、木、陶、布、蒸汽、颗粒、窗影。
5. **UI 层**：十类核心组件及其状态。
6. **信物层**：器物基底、铭文、稀有度、出处标签分开交付。
7. **文化内容层**：事实卡、创意酒签、双语分享卡。
8. **提交层**：主视觉、流程图、藏品效果图、演示封面。

生成模型只产无文字图像；标题、铭文、印章、编号和链上信息必须由前端或设计软件叠加。

---

## 10. 质量门槛

任何素材进入 P0 包前必须通过：

- 文化：没有未经核对的事实、服饰、器物和民俗细节。
- 视觉：色板、光源、像素密度、轮廓与三张风格锚一致。
- 技术：尺寸、透明通道、命名、压缩和切图正确。
- 可读：中英文均不溢出；低对比纹理不干扰正文。
- 可用：键盘焦点、触控尺寸、减弱动效与断网降级可用。
- 来源：生成参数、参考图、授权与人工修改均进入资产清单。
