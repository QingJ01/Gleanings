# 《拾遗》幕二至幕四可玩闭环设计

> 确认日期：2026-07-23  
> 状态：用户逐段确认  
> 范围：幕二《冬酿》、幕三《月子酒》、幕四《归》的纯前端可玩版本

## 1. 目标与边界

本阶段把第一章从“第一幕可玩”扩展为能够从开坛连续体验到最终徽章的完整 RPG 叙事闭环。

必须交付：

1. 幕二、幕三具有大于视口的可探索地图、碰撞、遮挡、NPC、任务与物件状态变化；
2. 记忆幕切换玩家身份：幕二控制少女太婆，幕三控制中年太婆；
3. 幕四回到林怡视角，回收前三幕选择并生成本地双语文化酒签；
4. 冬酿曲印、青花酒盏和“一坛回声”徽章进入章节收藏；
5. 章节级存档、安全检查点和第一幕 v1 存档迁移；
6. 从第一幕开坛到幕四结算的浏览器端完整路径。

本阶段不接入真实 LLM、钱包、voucher 服务或链上交易。AI 结果使用受控本地模板，铸造使用明确标注的演示状态；二者不得伪装成真实外部服务。

## 2. 架构

采用“共享章节状态 + 每幕独立场景”的混合结构。

- 共享层负责章节存档、选择路径、信物、文化卡、任务 HUD、对白、选择面板、幕间转场和章节结算；
- 每幕拥有独立地图内容、任务 reducer、交互物和演出逻辑；
- 通用地图几何、脚底碰撞、前景遮挡与玩家移动沿用第一幕已验证实现；
- 不在本阶段制作通用关卡编辑器，也不把三幕全部硬编码进一个大型场景。

章节状态以 `ChapterOneSaveV2` 为唯一持久化入口。第一幕原有 reducer 可以继续承担第一幕内部推进，但保存时映射进章节状态。载入旧 `gleanings.act1.save.v1` 时迁移为 v2，并保留感官选择、背包、完成状态和最近玩家位置。

## 3. 幕二《冬酿》

### 3.1 场景

- 地图：`44×30` tiles，原生 `1408×960`；
- 视口：`640×360`；
- 分区：入场区、阿凤师工位、曲房、陶坛区、拌曲台、下缸区、封坛区；
- 视觉：冬霜灰、陶褐、红曲朱；无渐变、环境光或伪 3D；
- 角色：玩家控制少女太婆，阿凤师为任务 NPC。

现有 `40×28` 酒坊地图作为构图与材质基础，扩展边缘与分区，不做简单拉伸。玩家和 NPC 不烘焙进背景。

### 3.2 主流程

```text
记忆醒来
  → 与阿凤师交谈
  → 检视红曲盘
  → 检视陶坛
  → 取得红曲样品
  → 霜降：拌红曲
  → 立冬：下缸
  → 冬至：封坛
  → 向阿凤师追问
  → 获得冬酿曲印
  → 进入幕三
```

三个工序没有失败判定。顺序不正确时只给出当前节气提示，不消耗物品。每完成一步，工位贴图、任务文本、旁白和进度标记同步变化。

核心选择：

- `ask_hongqu`：解锁《古田红曲》，普通曲印；
- `ask_winter`：解锁《冬酿与节气》，稀有曲印；
- `ask_future`：解锁隐藏卡；若第一幕选择“坛身的凉”，生成传世“留白印”，否则为稀有。

## 4. 幕三《月子酒》

### 4.1 场景

- 地图：`36×26` tiles，原生 `1152×832`；
- 分区：灶间、储物间、酒坛架、餐桌区、产妇休息区；
- 角色：玩家控制中年太婆，阿珍与家人为 NPC；
- 关键物件：碗、面线、老酒、灶台、成品面线、青花酒盏；
- 灶火只作为明确可见的实体阻挡，不造成伤害。

### 4.2 主流程

```text
多年后醒来
  → 与家人交谈
  → 取得碗
  → 取得面线
  → 舀取老酒
  → 回到灶台烹制
  → 盛好老酒面线
  → 交给阿珍
  → 回应这份人情
  → 获得青花酒盏
  → 返回现代
```

三件材料允许任意收集顺序。背包和任务 HUD 显示 `0/3` 到 `3/3`。烹制完成后，灶台、餐桌、阿珍和家人的位置改变，制造一次明确的场景状态回收。

核心选择：

- `receive_warmth`：铭文“暖”，结局基调温暖；
- `ask_burden`：铭文“承”，结局基调敬意；
- `watch_hands`：铭文“念”，结局基调怅惘。

## 5. 幕四《归》

幕四复用 `30×20` 公寓地图，不扩大地图。场景使用开坛后物件状态：酒坛已开、任务箭头移至米娅或酒签、纸箱任务关闭。

主流程：

```text
回到公寓
  → 与米娅交谈
  → 选择如何用英文解释老酒
  → 展开选择册页
  → 生成本地双语文化酒签
  → 接受或重试一次
  → 演示铸造
  → 获得“一坛回声”
  → 章节完成
```

文化酒签固定包含：

- 中文名与英文名；
- 一句海外读者可独立理解的英文说明；
- 明确标记为“文化事实”的只读短注；
- 明确标记为“创意表达”的玩家旅程短文；
- 第一幕感官、幕二追问、幕三铭文和幕四表达角度；
- 本地模板来源标记。

重试只允许一次，并在同一选择集合内切换表达模板，不改变文化事实。演示铸造不请求钱包，只生成本地 token 占位编号与 `pathHash` 展示。

## 6. 素材

复用：

- 第一幕林怡、米娅、公寓、HUD、对白、选择、转场；
- 现有酒坊地图、酒坊 tileset、少女太婆风格稿、红曲盘、红曲样品、冬酿曲印。

新增：

- `44×30` 无角色酒坊完整背景；
- 少女太婆与阿凤师 `96×192` 行走表；
- 三个酒坊工位的未完成/完成状态；
- `36×26` 灶间完整背景；
- 中年太婆、阿珍 `96×192` 行走表；
- 碗、面线、老酒、灶台状态、成品面线、青花酒盏；
- 章节徽章与酒签展示所需的本地组合层。

全部新增位图遵守固定 24 色、最近邻缩放、硬像素边缘和透明通道 `0/255`。关键素材加载失败时显示资源 ID；非关键装饰缺失时继续运行。

## 7. 存档与数据

章节存档至少包含：

```ts
type ChapterOneSaveV2 = {
  version: 2;
  currentAct: 1 | 2 | 3 | 4 | "complete";
  checkpoint: string;
  act1Sense: "aroma" | "hongqu_red" | "cold_clay" | null;
  act2Question: "ask_hongqu" | "ask_winter" | "ask_future" | null;
  act3Inscription: "warm" | "inherit" | "remember" | null;
  act4Explanation: "letter" | "three_generations" | null;
  inventory: string[];
  relics: string[];
  cultureCards: string[];
  playerTile: { x: number; y: number };
  labelRetryUsed: boolean;
  chapterComplete: boolean;
};
```

安全检查点：

- 幕二：取得样品、每个工序完成、获得曲印；
- 幕三：每个材料取得、烹制完成、获得酒盏；
- 幕四：完成解释选择、接受酒签、演示铸造完成。

## 8. 验收

1. 第一幕旧存档能进入幕二，不丢失感官选择；
2. 幕二、幕三地图大于视口，摄像机跟随、碰撞和遮挡正确；
3. 任务顺序、任意材料收集、选择和信物变体均有 reducer 测试；
4. 刷新可从所有安全检查点恢复；
5. 幕四能回收四次选择并生成稳定酒签；
6. 单元测试、类型检查、生产构建和完整 Playwright 路径全部通过；
7. 主流程不依赖网络、钱包、AI 或链上服务。
