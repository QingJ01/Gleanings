# 《拾遗》第一幕《开坛》可运行垂直切片设计

> 确认日期：2026-07-23
> 状态：用户已逐段确认
> 目标平台：桌面浏览器优先，移动端保留输入与布局接口

## 1. 目标与边界

第一幕在 3–5 分钟内完成五件事：

1. 教会移动、交互、任务、背包和选择；
2. 建立林怡身处海外、与家乡和太婆保持距离的状态；
3. 让玩家亲手找到字条、走到酒坛前并揭坛；
4. 记录玩家最先被香气、红色还是坛身温度吸引；
5. 以酒香进入第二幕冬日酒坊。

第一幕是可运行的纯 2D RPG，不是视觉小说背景。地图必须大于视口，有碰撞、
可选检视物、任务门槛、物件状态变化、背包和存档。

不在第一幕实现战斗、钱包、铸造、复杂解谜、自由文本输入或运行时 AI。

## 2. 人物命名

| 程序 ID | 中文显示名 | 第一幕职责 |
|---|---|---|
| `actor_yi` | 林怡 | 玩家角色 |
| `actor_mia` | 米娅 | 室友，推动林怡面对酒坛 |
| `taipo` | 太婆 | 通过字条和记忆出现 |

代码和存档使用稳定英文 ID，界面只显示中文人名。

## 3. 完整玩家流程

```text
进入公寓
  → 移动三格，完成移动教学
  → 自由检视房间
  → 打开纸箱，获得太婆的字条
  → 在背包中阅读字条
  → 米娅从门口入场
  → 走到酒坛前触发对话
  → 选择最先注意到的感官
  → 长按交互键揭开红布
  → 房间状态变化与酒香转场
  → 保存第一幕结果并进入冬日酒坊
```

主流程不能被可选检视物阻塞。玩家提前碰酒坛时只得到短提示，不会跳过字条。

## 4. 地图规格与布局

| 项目 | 规格 |
|---|---|
| 地图 | `30×20 tiles` |
| 像素尺寸 | `960×640` |
| tile | `32×32` |
| 视口 | `640×360` |
| 出生点 | `(14, 15)` |
| 摄像机 | 跟随林怡，整数像素，地图边缘停止 |

```text
┌──────────────窗户 / 远处城市──────────────┐
│ 工作桌                  旧照片          厨房 │
│                                            │
│ 纸箱区 ───────── 林怡出生点 ────────── 酒坛 │
│                                            │
│ 玄关                         米娅入场点 / 门 │
└────────────────────────────────────────────┘
```

| 对象 | tile 坐标 | 是否阻塞 | 作用 |
|---|---:|---:|---|
| 工作桌 | `(7, 5)` | 是 | 英文账单与未接来电 |
| 窗户 | `(15, 3)` | 是 | 海外城市氛围 |
| 旧照片 | `(13, 7)` | 是 | 太婆与家族提示 |
| 纸箱 | `(6, 13)` | 是 | 获得字条 |
| 酒坛 | `(25, 8)` | 是 | 主任务与揭坛 |
| 米娅入场点 | `(26, 16)` | 否 | 读信后生成 NPC |

地图层：

1. `ground`：地板、地毯；
2. `walls`：墙、窗、固定结构；
3. `furniture`：桌、沙发、料理台、纸箱；
4. `collision`：不可通行格；
5. `objects`：酒坛、字条、照片等状态物；
6. `above`：门框、吊灯和角色遮挡层；
7. `light`：窗光与台灯的硬边像素遮罩。

## 5. 任务、交互与引导

| 阶段 | 任务 ID | 任务条 | 完成条件 |
|---|---|---|---|
| 教学 | `act1_move` | 在房间里走走 | 移动累计 3 格 |
| 寻物 | `act1_find_box` | 整理太婆寄来的纸箱 | 与纸箱交互 |
| 阅读 | `act1_read_note` | 看看太婆留下的话 | 在背包打开字条 |
| 开坛 | `act1_find_jar` | 找到太婆留下的酒坛 | 靠近并检视酒坛 |
| 选择 | `act1_choose_sense` | 你最先注意到什么？ | 完成三选一 |
| 转场 | `act1_open_jar` | 揭开红布 | 长按交互键 0.6 秒 |

引导保持克制：

- 35 秒没有推进时，当前目标播放一次两帧微亮；
- 连续三次无效交互后，任务条显示方向提示；
- 交互范围为目标边缘外 `1 tile`；
- 可交互对象只有玩家朝向它时显示提示；
- 对话、背包、选择和转场期间锁定移动；
- 主流程对象允许重复检视，但副作用只能执行一次。

可选检视物：外卖盒、英文账单、未接来电、窗外城市、旧照片。它们只建立氛围，
不添加新系统，不阻塞任务。

## 6. 状态机与存档

```text
ARRIVE
  → EXPLORE
  → NOTE_ACQUIRED
  → NOTE_READ
  → MIA_ENTERED
  → JAR_INSPECTED
  → SENSE_CHOSEN
  → JAR_OPENING
  → COMPLETE
```

第一幕存档：

```ts
type Act1Save = {
  version: 1;
  phase:
    | "ARRIVE"
    | "EXPLORE"
    | "NOTE_ACQUIRED"
    | "NOTE_READ"
    | "MIA_ENTERED"
    | "JAR_INSPECTED"
    | "SENSE_CHOSEN"
    | "JAR_OPENING"
    | "COMPLETE";
  questId: string;
  inventory: string[];
  inspectedObjects: string[];
  senseChoice: "aroma" | "hongqu_red" | "cold_clay" | null;
  playerTile: { x: number; y: number };
  movementLocked: boolean;
  act1Complete: boolean;
};
```

安全检查点在获得字条、阅读字条、完成选择和揭坛开始时写入。
刷新后恢复最近安全阶段，不在对白句中间或长按交互中间恢复。

损坏或未知版本存档回退到 `ARRIVE`，同时保留一份原始字符串供调试。

## 7. 对白与选择

开场：

> 旁白：凌晨一点，窗外的城市仍亮着。
>
> 林怡：太婆寄来的最后一箱东西……我拖了三个月还没整理。

纸箱：

> 林怡：旧衣服、药袋，还有……一张字条。
>
> 太婆的字条：小怡，等你想家了，就开。——太婆
>
> 林怡：她知道我一定会把它带到这里。

米娅入场：

> 米娅：你还没睡？
>
> 林怡：在整理太婆留下的东西。
>
> 米娅：就是那坛一直没开的酒？
>
> 林怡：嗯。她让我“等想家了再开”。
>
> 米娅：那你现在想家吗？
>
> 林怡：……我不知道。

酒坛选择：

| 选择 | 存档值 | 即时反馈 | 后续倾向 |
|---|---|---|---|
| 那股香气 | `aroma` | 琥珀像素与暖色呼吸 | 温情 |
| 那一抹红 | `hongqu_red` | 红布、封泥与红曲颗粒变亮 | 工艺 |
| 坛身的凉 | `cold_clay` | 环境声变远，冷灰边缘收拢 | 时间 |

揭坛：

> 林怡：太婆，我开了。
>
> 米娅：林怡……你看见了吗？
>
> 旁白：酒香没有散开。它像一条路，把你拉进一场很久以前的冬天。

## 8. 技术架构

第一幕放入新的 `game/`：

- Vite + React + TypeScript；
- Phaser 3 负责游戏画布；
- Vitest 负责纯逻辑测试；
- Playwright 负责完整演示路径；
- JSON 驱动对白、任务、交互和地图数据；
- `localStorage` 保存版本化状态。

```text
game/src/
├─ app/
├─ game/
│  ├─ scenes/
│  │  ├─ BootScene.ts
│  │  ├─ ApartmentScene.ts
│  │  └─ MemoryTransitionScene.ts
│  └─ systems/
│     ├─ InteractionSystem.ts
│     ├─ QuestSystem.ts
│     ├─ DialogueSystem.ts
│     ├─ InventorySystem.ts
│     └─ SaveService.ts
├─ content/act1/
│  ├─ dialogue.zh-CN.json
│  ├─ quests.json
│  ├─ interactables.json
│  └─ apartment-map.json
└─ assets/act1/
```

Phaser 配置：

```ts
{
  width: 640,
  height: 360,
  pixelArt: true,
  roundPixels: true,
  backgroundColor: "#211A17",
  scale: {
    mode: Phaser.Scale.FIT,
    autoCenter: Phaser.Scale.CENTER_BOTH
  }
}
```

Phaser 内实现像素 HUD 与对白，保证坐标、字体和缩放统一。React 只负责画布外壳、
开始页、设置，以及后续钱包和铸造界面。

## 10. 输入、响应式与可访问性

| 动作 | 键位 |
|---|---|
| 移动 | WASD / 方向键 |
| 交互 / 确认 | E / Space / Enter |
| 背包 | I |
| 返回 | Esc |

- 第一版桌面优先；
- 移动端预留虚拟方向键与交互键，不阻塞桌面交付；
- 对话可逐字播放，也可一键显示整句；
- 支持关闭屏幕震动、降低闪烁与静音；
- 颜色不是唯一提示，交互对象同时使用轮廓和图标；
- 字体加载失败时使用本地中文回退字体，不能出现方框字符。

## 11. 错误处理

- 地图或关键素材加载失败：显示带文件 ID 的像素错误面板，不黑屏；
- 非关键音频失败：静音继续；
- 存档解析失败：回退新游戏并记录损坏内容；
- 任务事件重复触发：幂等处理，不重复获得字条或推进阶段；
- 玩家卡入碰撞体：恢复最近安全 tile；
- 第一幕结束但第二幕未实现：显示“冬酿记忆已解锁”的可返回完成页。

## 12. 验收标准

功能：

- 新游戏能从林怡出生完整走到揭坛转场；
- 字条进入背包且只能获得一次；
- 未读字条不能进入选择；
- 三个感官选择分别写入存档；
- 刷新后能从四个安全检查点恢复；
- 米娅按阶段出现并使用中文名；
- 第一幕结束能进入完成页或第二幕占位场景。

视觉与技术：

- 画布逻辑尺寸为 `640×360`；
- 公寓地图为 `30×20`，摄像机边缘正确；
- 所有位置和缩放为整数；
- 正式像素素材不超过 24 色；
- 透明资产无部分 alpha；
- 无抗锯齿、模糊、渐变或伪 3D；
- `npm test`、`npm run build` 和第一幕 Playwright 路径全部通过。
