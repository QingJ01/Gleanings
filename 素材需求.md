# 《拾遗 · Gleanings》视觉风格与素材需求

> 第一章《福建老酒》的生产级素材规范：tile map、角色行走表、场景物件、UI、信物、文化卡、音频、文案、生成记录与链上 metadata。
> 版本 v2.0 ｜更新：2026-07-23
> 对齐：[内容设计.md](内容设计.md) · [链上设计.md](链上设计.md) · [纯 2D RPG 设计](docs/plans/2026-07-23-pure-2d-rpg-design.md)

## 0. 生产原则、优先级与状态

### 0.1 四条生产原则

1. **先定锚，再批量生成**：酒坊实机视口、酒坊 tileset、Yi 行走表、原生 HUD 四张 v2 风格锚未通过，不生产后续地图。
2. **图像不含文字**：AI 只生成无文字的场景、人物、器物和纹理；标题、铭文、编号、印章与链上信息由前端或设计软件叠加。
3. **事实与创意分层**：已核对文化事实为只读输入；AI 只负责表达、组合和玩家个性化，不补写未知史实。
4. **游戏结构优先**：场景先做 tile map、碰撞、交互点和状态层，再做装饰；HUD、任务、背包和对白按原生像素网格实现。

### 0.2 优先级

- `P0`：完整 Demo 闭环必需，未完成则不能交付。
- `P1`：显著提升情绪、可分享性与评审表现。
- `P2`：冲刺加分，不阻塞主流程。

### 0.3 状态

`briefed → generating → visual_review → cultural_review → integration → ready`

任何包含器物、服饰、工艺与民俗的资产都必须经过 `cultural_review`；纯 UI 图标可跳过该状态。

## 1. 目录、命名与资产清单

```text
assets/
├─ rpg_v2/
│  ├─ palette/                    # 固定 24 色母板
│  ├─ previews/                   # 实机视口与 1920×1080 交付板
│  ├─ tilesets/                   # 256×256 / 512×512 图集
│  ├─ maps/                       # 大于视口的完整地图与遮挡层
│  ├─ sprites/                    # 3×4 四方向行走表
│  ├─ objects/                    # 32×32 / 64×64 场景物件
│  ├─ items/                      # 32×32 图标、128×128 详情
│  ├─ ui/                         # 640×360 原生像素 HUD
│  ├─ fx/                         # 允许局部半透明的特效
│  ├─ audio/
│  ├─ manifest/
│  └─ prompts/
├─ style/                         # v1 生成源图，仅供题材参考
├─ bg/                            # v1 背景，禁止与 v2 混用
├─ characters/                    # v1 立绘，禁止与 v2 混用
└─ items/base/                    # v1 器物源素材

metadata/                          # ERC-721 metadata
CREDITS.md                        # 授权与生成工具说明
```

命名规则：

- 文件名全部小写加下划线，如 `tileset_brewery_256x256.png`。
- 状态或变体放在末尾，如 `spr_yi_walk_winter_96x192.png`。
- 不使用 `final`、`new`、`v2_last`；版本写入 manifest。
- `asset_id`、剧情场景 ID、`pieceId` 与前端数据键必须一一对应。

`assets.manifest.json` 每项至少包含：

```json
{
  "asset_id": "spr_yi_walk",
  "file": "assets/rpg_v2/sprites/spr_yi_walk_96x192.png",
  "priority": "P0",
  "status": "visual_review",
  "width": 96,
  "height": 192,
  "grid": [3, 4],
  "cell": [32, 48],
  "source": "ai_plus_manual",
  "prompt_version": "rpg-style-v2.0",
  "seed": "record-if-supported",
  "references": [],
  "human_edits": "palette quantization, silhouette cleanup",
  "cultural_reviewer": null,
  "license": "project-owned-output"
```

| `lacquer-700` | 木框、漆面 | `#5B3A29` |
| `clay-500` | 陶坛与暖阴影 | `#7A5845` |
| `blue-500` | 文化出处、可信信息 | `#48677A` |
| `frost-300` | 冬日环境、锁定状态 | `#B7C2C0` |
| `straw-400` | 编号、稀有度辅助 | `#D4B46A` |

规则：

- 所有 tile、sprite、object 和 item 必须量化到
  [`assets/rpg_v2/palette/fujian_rpg_24.png`](assets/rpg_v2/palette/fujian_rpg_24.png)，单图不得超过 24 色。
- 红曲朱只用于玩家动作、焦点和选择印记。
- 青花蓝只用于文化事实、来源和可验证信息。
- 阴影使用墨褐与陶褐，不使用纯黑。
- 禁止把亮金作为主按钮色。

### 2.2 字体

| 角色 | 中文 | 英文 | 交付 |
|---|---|---|---|
| 展示标题 | Noto Serif SC / 思源宋体 | Fraunces | Web 字体或本地子集 |
| 正文 | MiSans / 系统人文黑体 | Geist | Web 字体或系统回退 |
| 编号/状态 | Fusion Pixel 12px | 同系列等宽像素体 | 仅短标签 |

需要记录每款字体的授权与加载方式。品牌主字体禁止使用 Inter、Roboto、Arial、Open Sans、Helvetica。

### 2.3 布局与间距

- 游戏原生画布固定 `640×360`；只允许 `2×`、`3×` 最近邻整数缩放。
- tile 固定 `32×32`；角色单帧固定 `32×48`；摄像机和角色落点取整数。
- 场景必须大于视口，摄像机跟随玩家；剧情演出可临时锁镜头。
- 基础间距：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。
- 桌面内容最大宽度：`1440px`，主舞台保持 16:9。
- 页面安全边距：桌面 `32–64px`，移动端 `16px`。
- 正文最小字号：桌面 `16px`，移动端 `15px`。
- 触控目标：不小于 `44×44px`。
- 主要容器采用“外层材质承托 + 内层内容纸面”的双层展框。

### 2.4 动效

| Token | 时长 | 用途 |
|---|---:|---|
| `motion_quick` | 180–220ms | 按压、图标、焦点 |
| `motion_standard` | 320–450ms | 酒签抬起、纸页展开 |
| `motion_narrative` | 1.2–1.8s | 入记忆、章节转场、藏品显影 |

统一缓动：`cubic-bezier(0.32, 0.72, 0, 1)`。只动画 `transform` 和 `opacity`。

## 4. 地图、tileset 与场景规模

`640×360` 是摄像机视口，不是地图尺寸。地图在 `32×32` 网格上构建，
画面只显示其中一部分；不得用一张 640×360 插画冒充可探索场景。

| ID | 地图 | 网格 / 像素 | 视口占比 | P0 内容 |
|---|---|---:|---:|---|
| `map_apartment_night` | 海外公寓·夜 | 30×20 / 960×640 | 1.67× | 开坛点、纸箱、门窗、Mia |
| `map_brewery_winter` | 冬日酒坊 | 40×28 / 1280×896 | 2.49× | 红曲盘、陶坛区、灶台、工人、出口 |
| `map_kitchen_yuezi` | 灶间·月子房 | 32×24 / 1024×768 | 2.13× | 灶火、鸡汤/面线、照护动线、家人 |

每张地图另交：

- `*_ground.png`：地面层；

## 8. UI 组件与状态

除图标、纹理和特殊角标外，组件用 HTML/CSS 构建。设计交付需提供桌面与移动稿、状态稿和文本极限测试。

| 组件 ID | P0 状态 | 关键要求 |
|---|---|---|
| `ui_story_stage` | 默认/互动热点/暂停 | 16:9；人物与热点不遮挡批注 |
| `ui_dialogue_sheet` | 旁白/人物/长英文 | 宣纸条；支持 2–4 行正文 |
| `ui_choice_tag` | 默认/悬停/焦点/已选/禁用 | 酒签抬起与压印；键盘焦点明显 |
| `ui_culture_note` | 收起/展开/待核对 | 青花蓝来源标记 |
| `ui_chapter_tab` | 当前/完成/锁定 | 幕、地点、年代三层信息 |
| `ui_lang_toggle` | 中文/EN | 不用国旗表示语言 |
| `ui_generation_sheet` | 准备/生成中/结果/降级 | 同屏区分事实与创意 |
| `ui_wallet_button` | 未连接/已连接/错误 | 不使用加密货币霓虹 |
| `ui_mint_panel` | 待签名/确认中/成功/失败 | 错误信息可读、可重试 |
| `ui_specimen_frame` | 锁定/已得/稀有/选中 | 双层展框；不做商城卡片 |
| `ui_provenance_mark` | 事实/AI/链上 | 颜色与文字双重区分 |
| `ui_share_card` | 中文/英文/双语 | 1200×630，文本不被平台裁切 |

必要图标：

`back / next / sound / mute / language / source / wallet / external_link / retry / share / close`

图标使用 1–1.5px 视觉重量或手工像素线，不使用粗重默认图标包。

## 10. 特效与转场

优先用 CSS、Canvas 或少量精灵图实现，避免长视频。

| ID | 实现 | 优先级 | 限制 |
|---|---|---|---|
| `fx_aroma_memory` | Canvas 颗粒+透明层 | P0 | 入记忆；粒子约 40–60 |
| `fx_choice_stamp` | CSS transform+朱印纹理 | P0 | 选择确认反馈 |
| `fx_item_reveal` | 像素遮罩+opacity | P0 | 信物显影 |
| `fx_mint_confirm` | 陶釉高光+纸页展开 | P0 | 不用烟花、金币 |
| `fx_snow` | Canvas/CSS | P1 | 幕二，减弱动效时关闭 |
| `fx_steam` | 低帧率精灵/Canvas | P1 | 幕三，不遮挡人物脸部 |
| `fx_page_transition` | transform+opacity | P1 | 收藏馆页切换 |

每个效果必须有 `prefers-reduced-motion` 降级：保留淡入和状态文字，关闭漂移、旋转和溶解。

## 14. AI 生成规范

### 14.1 全局提示词结构

```text
authentic low-resolution 16-bit top-down 3/4 narrative RPG,
orthographic 32x32 tile grid, 32x48 actor frame,
hard square pixel clusters, crisp 1-pixel edges,
flat stepped 2-3 tone shading, fixed 24-color earthy palette,
ink brown, wood, clay, hongqu red, ochre, parchment, cold blue-gray,
[subject], [era], [location], [action], [walkable path and interaction point],
no anti-aliasing, no gradient, no blur, no bloom, no soft glow,
no painterly texture, no 3D render, no depth of field,
no text, no letters, no watermark
```

负面约束：

```text
generic imperial China, palace, dragon, phoenix, lantern festival,
bright red and gold luxury, cyberpunk, neon, crypto dashboard,
glossy NFT card, 3D render, isometric diorama, photorealistic,
smooth vector, soft lighting, anti-aliasing, gradient, blur,
oversized portrait, random calligraphy, text, logo, watermark
```

### 14.2 分类补充

- 场景：必须写清地图网格、可行走通道、碰撞边界、交互点和摄像机视口。
- 人物：写清 3×4 排布、方向顺序、脚底基线、服装和动作；不靠随机 seed 维持一致。
- 器物：写清 32×32 / 64×64 占格、交互轮廓、材质和观看角度；不得直接生成铭文。
- tileset：必须逐格生成或逐格清理，禁止阴影和轮廓跨越格子边界。

### 14.3 生产流程

1. 冻结 `640×360` 画布、`32×32` tile、`32×48` actor 与固定 24 色。
2. 生成酒坊实机视口、tileset、Yi 行走表和原生 HUD 四张 v2 风格锚。
3. 逐格重排到目标网格，alpha 阈值化为 `0/255`。
4. 无抖动量化到项目色板，只做最近邻整数缩放。
5. 人工清理轮廓、无缝边缘、手、器物结构和文化错误。
6. 先完成碰撞、交互点和状态变化，再扩展装饰物件。
7. 文字、铭文、编号与来源标记由 UI 层独立叠加。
8. 更新 manifest、生成记录、授权与审核状态。
