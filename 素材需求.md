## 0. 生产原则、优先级与状态

### 0.1 四条生产原则

1. **先定锚，再批量生成**：酒坊场景、少女太婆、冬酿曲印、酒签选择四张风格锚未通过，不生产后续素材。
2. **图像不含文字**：AI 只生成无文字的场景、人物、器物和纹理；标题、铭文、编号、印章与链上信息由前端或设计软件叠加。
3. **事实与创意分层**：已核对文化事实为只读输入；AI 只负责表达、组合和玩家个性化，不补写未知史实。
4. **结构优先于位图**：卡框、按钮、状态面板优先用 CSS/HTML 组件；位图只提供纹理、器物和必要装饰，避免大量不可维护切图。

### 0.2 优先级

- `P0`：完整 Demo 闭环必需，未完成则不能交付。
- `P1`：显著提升情绪、可分享性与评审表现。
- `P2`：冲刺加分，不阻塞主流程。

### 0.3 状态

`briefed → generating → visual_review → cultural_review → integration → ready`

任何包含器物、服饰、工艺与民俗的资产都必须经过 `cultural_review`；纯 UI 图标可跳过该状态。

## 1. 目录、命名与资产清单

```text
assets/
├─ style/                         # 风格锚与色板
├─ bg/                            # 像素背景
├─ characters/                    # 人物基底与表情覆盖
├─ materials/                     # 纸、木、陶、布、颗粒等纹理
├─ items/
│  ├─ base/                       # 器物基底
│  ├─ inscriptions/               # 铭文透明层
│  ├─ rarity/                     # 稀有度视觉层
│  └─ frames/                     # 藏品展示框
├─ cards/                         # 文化卡插画与分享卡底图
├─ ui/                            # UI 必要纹理、角标和图标
├─ fx/                            # 必须使用位图的叠层
├─ audio/
│  ├─ bgm/
│  ├─ ambience/
│  └─ sfx/
├─ generated/
│  └─ fallback/                   # 实时生成失败时的预生成结果
├─ copy/                          # i18n、文化事实、生成模板
└─ deliverables/                  # 1920×1080 主视觉与提交物

metadata/                          # ERC-721 metadata
manifest/assets.manifest.json     # 尺寸、来源、版本、状态
CREDITS.md                        # 授权与生成工具说明
```

命名规则：

- 文件名全部小写加下划线，如 `bg_brewery_winter.png`。
- 状态或变体放在末尾，如 `ch_taipo_young_warm.png`。
- 不使用 `final`、`new`、`v2_last`；版本写入 manifest。
- `asset_id`、剧情场景 ID、`pieceId` 与前端数据键必须一一对应。

`assets.manifest.json` 每项至少包含：

```json
{
  "asset_id": "it_relic_dongniang",
  "file": "assets/items/base/it_relic_dongniang.png",
  "priority": "P0",
  "status": "visual_review",
  "width": 1024,
  "height": 1024,
  "source": "ai_plus_manual",
  "prompt_version": "style-v1.1",
  "seed": "record-if-supported",
  "references": [],
  "human_edits": "palette quantization, silhouette cleanup",
  "cultural_reviewer": null,
  "license": "project-owned-output"
}
```

## 2. 视觉 Token

### 2.1 色板

| Token | 用途 | HEX |
|---|---|---|
| `ink-950` | 主背景、最深文字 | `#211A17` |
| `ink-700` | 次级文字、柔和阴影 | `#5B514A` |
| `paper-100` | 主纸面 | `#EADDC5` |
| `paper-50` | 高亮纸面 | `#F4EBDD` |
| `amber-500` | 酒液、记忆高光 | `#C9873F` |
| `hongqu-600` | 选择、印记、关键动作 | `#A83B32` |
| `lacquer-700` | 木框、漆面 | `#5B3A29` |
| `clay-500` | 陶坛与暖阴影 | `#7A5845` |
| `blue-500` | 文化出处、可信信息 | `#48677A` |
| `frost-300` | 冬日环境、锁定状态 | `#B7C2C0` |
| `straw-400` | 编号、稀有度辅助 | `#D4B46A` |

规则：

- 红曲朱只用于玩家动作、焦点和选择印记。
- 青花蓝只用于文化事实、来源和可验证信息。
- 阴影使用墨褐与陶褐，不使用纯黑。
- 禁止把亮金作为主按钮色。

### 2.2 字体

| 角色 | 中文 | 英文 | 交付 |
|---|---|---|---|
| 展示标题 | Noto Serif SC / 思源宋体 | Fraunces | Web 字体或本地子集 |
| 正文 | MiSans / 系统人文黑体 | Geist | Web 字体或系统回退 |
| 编号/状态 | Fusion Pixel 12px | 同系列等宽像素体 | 仅短标签 |

需要记录每款字体的授权与加载方式。品牌主字体禁止使用 Inter、Roboto、Arial、Open Sans、Helvetica。

### 2.3 布局与间距

- 基础间距：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。
- 桌面内容最大宽度：`1440px`，主舞台保持 16:9。
- 页面安全边距：桌面 `32–64px`，移动端 `16px`。
- 正文最小字号：桌面 `16px`，移动端 `15px`。
- 触控目标：不小于 `44×44px`。
- 主要容器采用“外层材质承托 + 内层内容纸面”的双层展框。

### 2.4 动效

| Token | 时长 | 用途 |
|---|---:|---|
| `motion_quick` | 180–220ms | 按压、图标、焦点 |
| `motion_standard` | 320–450ms | 酒签抬起、纸页展开 |
| `motion_narrative` | 1.2–1.8s | 入记忆、章节转场、藏品显影 |

统一缓动：`cubic-bezier(0.32, 0.72, 0, 1)`。只动画 `transform` 和 `opacity`。


英文验收重点：

- 不把福建老酒直接等同于 grape red wine；
- 亲属称谓第一次出现时自然解释；
- 不使用医疗功效承诺；
- 不把 Mia 写成“外国人代表”或只负责赞叹；
- 语句适合海外年轻人阅读，不保留中式宣传腔。

## 13. ERC-721 Metadata 与存储

| 交付物 | 优先级 | 要求 |

| 信物主图 3 张 | P0 | 1024² PNG/WebP；无长文字 |

## 14. AI 生成规范

### 14.1 全局提示词结构

```text
editorial pixel art for an interactive Fujian folk-life archive,
tactile specimen-book presentation, restrained warm nostalgia,
limited palette: ink brown, rice paper, aged-wine amber, hongqu red,
lacquer wood, clay, blue-and-white porcelain accent, winter frost,
clean readable silhouettes, consistent upper-right warm light,
medium-low pixel density, lived-in domestic detail,
[subject], [era], [location], [action], [composition and safe area],
no text, no letters, no watermark
```

负面约束：

```text
generic imperial China, palace, dragon, phoenix, lantern festival,
bright red and gold luxury, cyberpunk, neon, crypto dashboard,
glossy NFT card, 3D render, photorealistic, smooth vector,
chibi anime, oversized eyes, random calligraphy, text, logo, watermark
```

### 14.2 分类补充

- 场景：必须写清年代、生活动作、光源、前中后景和 UI 留白。
- 人物：使用角色锚图、服装表、年龄与手部动作；不靠随机 seed 维持一致。
- 器物：写清材质、磨损、轮廓和观看角度；不得直接生成铭文。
- 材质：要求可平铺、低对比、无强方向性。

### 14.3 生产流程

1. 生成四张风格锚并锁版。
2. 创建色板文件和角色比例图。
3. 使用同一参考图与提示词版本批量生成。
4. 降采样到目标像素密度并量化到项目色板。
5. 人工清理轮廓、手、器物结构和文化错误。
6. 文字、铭文、编号与来源标记独立叠加。
7. 输出目标尺寸、缩略图和 Web 压缩版本。
8. 更新 manifest、生成记录、授权与审核状态。
